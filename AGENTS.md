# Agent Guidelines

## File Reading Policy

### Primary Principle
- **Avoid reading unrelated project files** by default
- Only read files that are directly relevant to the current task or query
-  Focus on the core project structure and active codebase

### When to Read Additional Files
- **Directly Requested**: User explicitly asks to read or check a file
- **Task-Critical**: The file is essential to complete the current task

### Necessity Check for Unrelated Files
Before reading any file that seems unrelated to the current task:
- Ask yourself: **"Is this file context absolutely necessary to complete the current task?"**
- Check if the information can be obtained from related/core files first

## Implementation Standards
- Minimize agent token utilization by avoiding unnecessary file reads
